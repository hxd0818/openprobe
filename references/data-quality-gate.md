# 数据质量门禁规范（v9.3 — 时效性强制门槛 + 缺口处理 + 推算闭环）

> **目的**：确保每份调查报告的数据质量可量化、可追溯、无断裂。
> **核心问题**：数据不够新怎么办？找不到数据怎么办？推算怎么才算合规？
> **⭐v9.3 核心变革**：从"事后标注型"升级为**"事前预防型"**——不再允许先搜到旧数据再用标签补救，
> 而是**强制要求优先获取上年度最新数据**，旧数据仅在穷尽最新数据搜索努力后才可降级使用。
> **生效时机**：与 `pir-template.md`（收敛）、`output-specs.md`（产出）、`analyst-identity.md`（身份）共同构成完整质量体系。
> **依赖关系**：本文件是 v9 质量体系的**第四根支柱**，前三根为 PIR驱动收敛 / 产出物规范 / 分析师职业规范。

## 二、数据时效性分级（Data Currency Classification）

### 2.1 四级时效定义

```
┌──────────────────────────────────────────────────────────────────┐
│                     📅 数据新鲜度等级                             │
├──────────┬─────────────┬────────────────────────────────────────┤
│   等级    │   时效阈值    │  定义、示例与使用规则                   │
├──────────┼─────────────┼────────────────────────────────────────┤
│                                                          │
│  🟢 FRESH │  ≤ 6个月    │ 当前财年/季度/月度数据                    │
│           │             │ 示例：2025全年财报(当前为2026.04)          │
│           │             │ ✅ 可用于所有类型判断，无需额外标注          │
│           │             │                                          │
│  🟡 CURRENT│  6-18个月   │ 去年完整财年数据                          │
│           │             │ 示例：2024全年营收、2025H1数据              │
│           │             │ ✅ 可作为基准数据用于定量判断               │
│           │             │ ⚠️ 跨实体对比时必须标注时间差异              │
│           │             │                                          │
│  🟠 STALE  │  18-36个月  │ 较旧数据                                 │
│           │             │ 示例：2023年前10月数据、2022年报            │
│           │             │ ⚠️ **不可直接用于当前判断**               │
│           │             │ 仅允许作为推算基准值（Level 3, §2.5.A）    │
│           │             │ 必须配合推算因子(增速/规模变化等)映射到当前  │
│           │             │ 输出格式：~X(按Y%从{旧基准}推算,2025E) 🟠   │
│           │             │ 可信度上限 0.65                             │
│           │             │                                          │
│  🔴 OBSOLETE│ > 36个月  │ 过期数据                                 │
│           │             │ 示例：2021年及之前的数据                    │
│           │             │ ⚠️ **不可直接用于任何判断**                │
│           │             │ 仅允许作为"历史基线"或Level 3推算基准       │
│           │             │ 需配合更强的推算因子+更宽不确定性区间        │
│           │             │ 可信度上限 0.50                             │
│           │             │                                          │
└──────────┴─────────────┴────────────────────────────────────────┘
```

### 2.2 时效等级判定规则

```
判定公式：
  data_age = current_date - data_period_end_date

  其中：
    current_date = 报告生成日期
    data_period_end_date = 数据覆盖期的结束日

  示例（报告日期=2026-04-07）：
    2025全年财报 → age ≈ 4个月 → 🟢 FRESH
    2024全年财报 → age ≈ 16个月 → 🟡 CURRENT
    2023前10月   → age ≈ 30个月 → 🟠 STALE
    2021年成立信息 → age > 48个月 → 🔴 OBSOLETE (但基础事实类可豁免)
```

### 2.3 基础事实豁免

以下类型的数据**不受时效分级约束**（因为不会随时间变化）：

| 豁免类型 | 示例 | 处理方式 |
|---------|------|---------|
| 法定注册信息 | 成立日期、法人、注册资本、统一社会信用代码 | 直接使用，标注"N/A—静态信息" |
| 固定背景信息 | 创始人学历、过往职务经历（已完成的事件） | 同上 |
| 产品技术参数 | 已发布产品的规格指标 | 标注产品发布日期即可 |
| 历史事件 | 过往融资轮次、并购记录 | 作为"历史事实"引用，不做当前状态判断 |

### 2.4 强制标注规则

**报告中的每个数值型数据点必须附带时效标记**。格式：

```markdown
# 行内简标（正文推荐）
公司2024年营收2.1亿元🟡（来源：企查查）

# 表格列标注（对比表必须）
| 企业 | 营收 | 时效 | 数据期间 | 来源 |
|------|------|------|---------|------|
| A公司 | 2.1亿 | 🟡 | 2024全年 | 企查查 |
| B公司 | 1.5亿 | 🟠 | 2023前10月 | 招股书 |
```

## 二点五、数据时效性强制门槛（⭐v9.3 新增 — 事前预防机制）

> **这是 v9.3 最核心的变更。** 此前的版本只定义了"如何标注旧数据"，但没有规定
> **"必须先尝试获取新数据才能接受旧数据"**。这导致报告中大量出现用2-3年前的旧数据
> 做当前判断的情况（如 KK集团报告使用2023前10月招股书数据作为核心财务指标）。
>
> **本节所有规则为强制性门槛，违反任一条 = 该数据点不得用于定量判断。**

### 2.5.A "最新优先"铁律（Latest-First Rule）

```
┌─ 数据获取强制流程（三级递进，逐级降级）──────────────┐
│                                                  │
│  目标：获取目标数据点 D 的"上年度"值               │
│  例："KK集团2025年度营收"、"泛联信息2025员工数"     │
│                                                  │
│  ═══ Level 1: 直接获取最新年完整数据 ═══           │
│  Step 1: 计算上年度时间窗口                        │
│    上年度 = 报告年份的前一个完整财年               │
│    例：2026年的报告 → 上年度 = 2025全年            │
│    接受范围：2025全年 / 2025H2 / 2025Q4 均可        │
│                                                  │
│  Step 2: 带时间锚定搜索（⚠️ 强制，见 §2.5.B）      │
│    执行至少 2 次不同组合的上年度搜索 query         │
│                                                  │
│  Step 3: 判断                                    │
│    ├─ 找到 🟢/🟡 完整数据 → 直接使用 ✅ 结束       │
│    └─ 未找到完整数据 → 进入 Level 2               │
│                                                  │
│  ═══ Level 2: 碎片新数据推算 ═══                  │
│  Step 4: 搜索上年度碎片数据                       │
│    碎片类型：部分期间数据(前N月/Q1-Q3)、同比增速、  │
│           同行对比提及、公司公告片段、新闻零星数据  │
│    搜索策略：                                     │
│    □ "{公司} 增长 2025" / "{公司} 增速 2024 2025"  │
│    □ "{公司} {指标} 上半年 2025"                   │
│    □ "{行业} 增长 2025" （找同行增速做锚定）        │
│    □ "{公司} 最新 动态 营收 利润"                  │
│    至少执行 3 次碎片数据搜索                      │
│                                                  │
│  Step 5: 判断                                    │
│    ├─ 获取到可用碎片 → 用推算方法（§四）推算完整值 │
│    │   输出：~X(推算方法, 碎片来源) 🟡            │
│    │   → ✅ 结束                                 │
│    └─ 无任何碎片 → 进入 Level 3                   │
│                                                  │
│  ═══ Level 3: 旧数据推算新数据 ═══                │
│  Step 6: 确认已有历史完整数据（🟠 STALE 级别）     │
│    ⚠️ 此数据本身不可直接用于当前判断！            │
│    仅作为推算的"基准值"                           │
│                                                  │
│  Step 7: 寻找推算因子                             │
│    需要至少一个"桥梁信息"将旧数据映射到当前：       │
│    □ 行业/公司增速指引（CAGR/同比/公司表述）       │
│    □ 门店数/员工数等规模指标的同期变化率           │
│    □ 同行可比公司的同期表现作为外部锚定            │
│    □ 宏观环境变化（行业整体增速/政策影响）         │
│                                                  │
│  Step 8: 判断                                    │
│    ├─ 有推算因子 → 用旧数据+因子推算最新年估值     │
│    │   输出：~X(按Y%增速从{旧基准}推算,2025E) 🟠  │
│    │   ⚠️ 可信度上限 0.65（低于碎片推算的0.85）   │
│    │   → ✅ 结束                                 │
│    └─ 无任何推算因子 → 进入 Level 4               │
│                                                  │
│  ═══ Level 4: 完全缺口 ═══                       │
│  Step 9: 执行完全缺口影响评估（§3.2 Step 4b）     │
│    不允许写"未获取到"！必须输出影响评估。          │
│    该 PIR confidence 上限设为 0.60（降级通过⚠️）   │
│                                                  │
│  ═══════════════════════════════                 │
│  ❌ 以下行为 = 严重方法论违规：                    │
│  ✗ 跳过 Level 1-2 直接使用旧数据作为当前值       │
│  ✗ 将旧数据（🟠）当最新数据用于定量判断           │
│  ✗ 用旧数据与🟢/🟡数据做精确比值计算              │
│  ✗ 未经推算直接写"据最新数据XX"但实际引用的是旧值 │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 2.5.B 时间锚定 Query 设计规范

> **所有涉及量化数据的搜索 query 必须包含时间锚定。**
> 这是"最新优先"铁律的执行抓手——如果 query 不带年份，
> 搜索引擎返回的结果排序权重偏向历史高流量内容（旧新闻/旧招股书），
> 导致 Agent 拿到旧数据却误以为是最新。

#### B1：Query 时间锚定模板

```
┌─ 必须遵守的 query 格式 ───────────────────────────┐
│                                                  │
│  基础模板：                                      │
│    "{实体名} {指标} {年份}"                       │
│                                                  │
|  ✅ 正确示例（带时间锚定）：                      │
│    "KK集团 营收 2025"                            │
│    "KK集团 营收 2024"                             │
│    "泛联信息 员工人数 2025"                       │
│    "名创优品 门店数量 2025"                       │
│    "THE COLORIST调色师 闭店 2024 2025"           │
│                                                  │
│  ❌ 错误示例（无时间锚定）：                      │
│    "KK集团 营收 利润"        → 可能返回2023招股书  │
│    "泛联信息 团队规模"       → 可能返回2022年数据  │
│    "名创优品 门店数"         → 可能返回2021年数据  │
│                                                  │
│  多年回溯模板（当最新年搜不到时）：                │
│    "{实体名} {指标} 2024 2025"                    │
│    "{实体名} 年报 2024 2025"                      │
│    "{实体名} 最新 营收 利润"  ("最新"作为兜底词）   │
│                                                  │
└──────────────────────────────────────────────────┘
```

#### B2：时间锚定与 Q1-Q4 铁律的关系

| 铁律 | 与时间锚定的关系 |
|------|----------------|
| **Q1（≤5词）** | 年份占1-2词，剩余3词用于指标描述 |
| **Q2（一问一搜）** | 一个query = 一个指标 + 时间锚定 |
| **Q3（精确/泛化）** | 精确搜索必须带年份；泛化搜索可用"最新""排名"等时间敏感词 |
| **Q4（3-6次/轮）** | 其中至少 **2 次** 必须是带时间锚定的量化数据搜索 |

#### B3：Agent 自检清单（每次设计 query 后执行）

```
□ 本次搜索是否涉及量化数据？（营收/利润/门店/员工/市占率等）
  └─ 是 → query 中是否包含以下至少一种时间锚定？
         □ 具体年份（2024/2025）
         □ "最新""当年""上年"
         □ "年报""财报""半年报"
  └─ 否（纯定性搜索）→ 无需时间锚定
```

### 2.5.C PIR 时效折扣与推算等级系数（⭐ 立即生效）

> **v9.3 正式启用。** 推算数据的折扣不仅看时间，还看推算方法层级。

```
原始置信度（基于证据强度）× 推算等级系数 = 最终有效置信度

┌─────────────┬──────────┬──────────────────────────────────────┐
│ 数据类型     │  系数     │  说明                               │
├─────────────┼──────────┼──────────────────────────────────────┤
│ 🟢 FRESH    │  ×1.00   │  直接获取的最新年完整数据             │
│ 🟡 CURRENT  │  ×0.95   │  去年完整财年数据                    │
│ ~ 碎片推算   │  ×0.85   │  Level 2: 用新碎片推算（§2.5.A L2）  │
│ ~ 旧数据推算 │  ×0.65   │  Level 3: 用旧数据+因子推算（L3）     │
│ 🔴 OBSOLETE │  ×0.40   │  仅历史基线，几乎不可用于当前判断     │
└─────────────┴──────────┴──────────────────────────────────────┘

⚠️ 关键变化（vs v9.2）：
  1. 新增"~碎片推算"等级(×0.85)——鼓励用碎片拼出新数据
  2. 🟠 STALE 不再是独立等级——旧数据必须经推算后才可用，归入"~旧数据推算"
  3. 🔴 从0.50降至0.40（更严格）
  4. 此折扣自动应用于所有 PIR 置信度计算

示例（以 KK集团为反面教材 → 正确做法对比）：

  ❌ v9.2 错误做法：
    PIR-BASIS = 0.92 （直接用2023前10月招股书数据）
    → 虚高！数据已30个月 old

  ✅ v9.3 正确做法（四级递进）：
    L1: 搜"KK集团 营收 2025"/"KK集团 营收 2024" → 无完整数据
    L2: 搜"KK集团 增长 2025"/"KK集团 闭店 2024 2025"
        → 发现碎片：行业增速8%+闭店10%+新闻提及"营收持平"
    → 推算：~52亿(碎片综合调整) 🟡
    → PIR-BASIS effective = 0.85(证据强度) × 0.85(碎片推算) = 0.72
    → 仍不满足 0.85 目标 → 继续搜索或接受降级

    若 L2 也无碎片：
    L3: 以招股书47.7亿(2023前10月)为基准
        推算因子：四度IPO失效+闭店潮+行业放缓 → 估-6%CAGR
    → ~42亿(按-6%从47.7亿推算,2025E) 🟠
    → PIR-BASIS effective = 0.85 × 0.65 = 0.55
    → 远低于目标 → 该PIR为数据质量受限项 ⚠️
```

强制执行位置：methodology.md Step ⑥（更新PIR置信度）

### 2.5.D 推算值强制声明模板

> 当数据经过 Level 2（碎片推算）或 Level 3（旧数据推算）后，**必须**使用以下格式：

```markdown
**Level 2 碎片推算声明**：
~[推算值]([推算方法], 碎片来源: [具体碎片数据及出处]) 🟡
例：KK集团2025年营收 ~52亿(按2023前10月47.7亿为基数,
   结合"2024年行业增速约8%+公司闭店约10%"综合调整,
   碎片源:新闻提及"2024营收持平"/招股书2023前10月)

**Level 3 旧数据推算声明**：
~[推算值](按[X]%从{旧基准[期间]}推算, [年份]E,
   推算因子: [桥梁信息来源]) 🟠
例：KK集团2025年营收 ~45亿(按-6%CAGR从招股书47.7亿/2023前10月推算, 2025E,
   推算因子: 四度IPO失效+闭店潮+行业增速放缓,
   ⚠️ 可信度上限0.65, 不确定性区间±30%)

**Level 4 完全缺口声明**：
⚠️ [数据名称] 未能从公开渠道获取。
经完整四级递进搜索（L1直接搜索N次 + L2碎片搜索N次 +
L3推算因子搜索N次），确认该数据点在OSINT范围内不可通过推算获得。
缺失影响：[具体说明对哪些判断造成限制]。
```

> **❌ 以下写法已全部禁止**：
> - "KK集团营收47.7亿元"（无时间标记、无推算标记、直接用旧值）
> - "据最新数据显示"（实际引用的是旧数据）
> - 用🟠/≈值与🟢值算精确比值而不标注不确定性

### 2.5.E 违规判定与处理

| 违规类型 | 判定标准 | 处理方式 |
|---------|---------|---------|
| **V-1：零次最新年搜索直接用旧数据** | 量化数据点无任何带年份锚定的搜索记录 | **该数据点无效**，必须从 Level 1 重做 |
| **V-2：跳过碎片推算直接降级旧数据** | 未执行 Level 2（碎片数据≥3次搜索）就进入 Level 3 | **违规**，必须补搜碎片数据 |
| **V-3：旧数据未经推算直接当当前值用** | 报告中出现🟠数据但无推算标记(~)和说明 | **严重违规**，重写相关章节 |
| **V-4：🟠/≈推算值与🟢/🟡数据精确混比** | 用低可信度推算值算"A是B的N倍"且未说明不确定性 | 改为定性描述+量级差异 |
| **V-5：PIR未施时效折扣** | 基于🟠/≈数据的PIR置信度未乘以时效系数 | 重算置信度，可能导致PIR降级 |
| **V-6：选择性使用有利年份** | 同一指标有多年数据却选最旧的展示 | 展示完整时间序列 + 推算值 |

### 2.5.F 三级递进速查卡（Agent 执行时贴身参考）

```
需要数据 D 的上年度值：
│
├─ L1: 搜 "{D} 2025" → 有？→ ✅ 直接用
│
├─ L1无 → L2: 碎片 hunt
│   "D 增长 2025" / "D 上半年 2025" / "D 最新" / "行业增速"
│   → 有碎片？→ ~推算(方法,碎片源) 🟡 ✅
│
├─ L2无 → L3: 旧数据+推算因子
│   有历史基准？有增速/规模变化等桥梁信息？
│   → 有？→ ~推算(按X%从{旧基准}推算,2025E) 🟠 ✅
│
└─ L3无 → L4: 完全缺口影响评估 ⚠️ 降级通过
```

## 三、缺口处理决策树（Data Gap Decision Tree）⭐核心

### 3.1 总原则

> **"不知道"不是可接受的答案。找不到精确数据时，Agent 必须通过合理的分析路径给出带置信度区间的估算或明确的影响评估。**

### 3.2 决策树

```
需要某项关键数据点 D（如"泛联信息2025年营收"）
│
├── Step 1: 能找到直接数据吗？
│   ├── YES → 使用直接数据，标注时效等级 → 结束 ✅
│   └── NO  → Step 2
│
├── Step 2: 能找到同期部分数据吗？
│   定义：同一时间段但不完整的数据
│   示例：需要全年 → 有前9月；需要Q4 → 有Q1-Q3汇总
│   ├── YES → 进入【推算路径A：部分数据补全】→ Step 3A
│   └── NO  → Step 2B
│
├── Step 2B: 能找到目标公司的历史序列吗？
│   定义：往年的同项数据（形成时间序列）
│   示例：没有2025营收，但有2023+2024营收
│   ├── YES → 进入【推算路径B：趋势外推】→ Step 3B
│   └── NO  → Step 2C
│
├── Step 2C: 能找到同行/竞对的同类数据吗？
│   定义：同行业可比公司的同期数据
│   示例：没有泛联营收，但有XSKY/杉岩的营收
│   ├── YES → 进入【推算路径C：锚定推断】→ Step 3C
│   └── NO  → Step 4（完全缺口）
│
├── Step 3A: 部分数据补全
│   可用方法：
│   ├── 年化外推（已知前N月）→ 见四、方法表 #1
│   ├── 季度比例法（已知前N季 + 往年季节性）→ 方法表 #1变体
│   └── 半年度插值（已知H1 + 往年H2/H1比）→ 方法表 #1变体
│   输出：~估值(方法说明) + 可信度上限
│   → 结束 ✅
│
├── Step 3B: 趋势外推
│   可用方法：
│   ├── 线性外推（稳定增长期）→ 方法表 #6
│   ├── CAGR延续（复合增速稳定）→ 方法表 #2
│   └── 增速衰减模型（增速逐年放缓）→ 方法表 #6变体
│   ⚠️ 外推跨度不超过已知序列的 50%
│   例：只有2年数据，最多外推1年
│   输出：~估值(按N年趋势外推) + 可信度上限
│   → 结束 ✅
│
├── Step 3C: 锚定推断
│   可用方法：
│   ├── 同行中位数锚定（行业均值）→ 方法表 #4
│   ├── 规模对标锚定（员工数/门店数/融资阶段）→ 方法表 #4变体
│   └── 市占率反推（知市场总规模+估市占率）→ 方法表 #3
│   ⚠️ 此路径不确定性最高，必须用 ≈ 标记
│   输出：≈估值(参照基准,不确定性高) + 低可信度
│   → 结束 ✅
│
└── Step 4: 完全缺口（所有路径均无数据）
    不允许写"未获取到"！必须执行：
    │
    ├── 4a: 评估该数据点的重要性
    │   ├── 核心（影响总体判断）→ 执行 4b
    │   └── 辅助（补充细节）→ 标注"缺失，不影响核心结论"
    │
    └── 4b: 完全缺口影响评估（核心数据专用）
        在报告中写入：
        "⚠️ [数据名称] 未能从公开渠道获取。
         经多角度搜索（列举尝试过的搜索策略），确认该数据点在OSINT范围内不可得。
         缺失影响：[具体说明对哪些判断造成限制]。
         建议：如有更高权限数据源（如商业数据库/内部资料），可优先补充此项。"
        
        同时在 status.json 的 gap_items 中记录，
        并将该 PIR 的 confidence 上限设为 0.60（因核心数据缺失）
        → 结束（降级通过）⚠️
```

### 3.3 决策树速查卡（Agent 执行时可参考）

```
找数据 → 有？→ 用
        → 无 → 有部分同期？→ 推算(A)
                → 无 → 有历史序列？→ 推算(B)
                        → 无 → 有同行数据？→ 推算(C)
                                → 无 → 影响评估(降级)
```

## 四、标准化推算规范（Standardized Estimation Protocol）

### 4.1 六种推算方法完整定义

| # | 方法名 | 适用场景 | 计算方式 | 标注格式 | 可信度上限 | 时效标记 |
|---|--------|---------|---------|---------|-----------|---------|
| **1** | **年化外推** | 已知前N月(N≥6)的部分年数据 | `值 × 12/N` | `~X(年化,按M月×12/M)` | 0.85 | 🟡 |
| **1a** | **季节性调整年化** | 有明显淡旺季的行业 | 年化值 × 季节系数 | `~X(年化,季节性调整,系数α)` | 0.80 | 🟡 |
| **2** | **CAGR反推** | 已知多年CAGR和基准年数据 | `基准值 × (1+CAGR)^N` | `~X(按CAGR Y%反推,Z年E)` | 0.70 | 🟠 |
| **3** | **比例缩放** | 已知总量及目标占比 | `总量 × 占比%` | `~X(按Y%占比推算)` | 0.80 | 🟡 |
| **3a** | **市占率反推** | 已知市场规模+估市占率 | `市场规模 × 估计市占率%` | `≈X(按市场总规模Z亿,估市占率Y%,不确定性高)` | 0.55 | 🟠 |
| **4** | **同行锚定** | 完全无数据，有同行参照 | 参照同行 × 调整系数 | `≈X(参照同行A的Y,不确定性强)` | 0.55 | 🟠 |
| **4a** | **规模因子锚定** | 有间接规模指标(员工/门店/融资) | 按规模因子折算同行数据 | `≈X(按员工数N为同行A的Y倍推算)` | 0.50 | 🟠 |
| **5** | **多源交叉取中位** | 多源数据矛盾或不完整 | 取各源中位数/加权平均 | `~X(三源交叉:A=a/B=b/C=c,取中位)` | 0.90 | 🟢 |
| **6** | **趋势外推** | 有2年以上历史序列 | 线性/指数/多项式拟合外推 | `~X(按N年趋势线性外推)` | 0.65 | 🟠 |

### 4.2 推算方法的选用优先级

```
当多种方法都可用时，按此顺序选择（可信度从高到低）：

  多源交叉(5) > 年化外推(1) > 比例缩放(3) > CAGR反推(2) > 趋势外推(6) > 同行锚定(4)

理由：
- 多源交叉 = 最接近"真实数据"（多方印证）
- 年化外推 = 基于实际数据的最小幅度加工
- 同行锚定 = 纯粹的间接推断，不确定性最大
```

### 4.3 推算禁止清单（硬性红线）

| # | 禁止行为 | 原因 | 发现后处理 |
|---|---------|------|-----------|
| E-E1 | **二次推算**：用推算值再做进一步推算 | 误差放大效应 | 改用原始数据重新推算，或降级为定性描述 |
| E-E2 | **推算值不给原始值**：只写估算不写依据 | 无法验证 | 补充原始值和方法说明 |
| E-E3 | **高不确定性给高精度**：🟠级推算写到小数点后多位 | 虚假精度 | 按可信度等级修约到合理位数 |
| E-E4 | **低可信度推算做跨实体精确对比**：用🟠/≈值算"A是B的N倍" | 误导读者 | 改为定性描述 + 标注量级差异 |
| E-E5 | **超长跨度外推**：外推时长超过已知序列50% | 外推误差随距离爆炸 | 截断到安全范围或改用同行锚定 |
| E-E6 | **选择性使用推算方向**：只选有利于预设结论的推算结果 | 先入为主偏差 | 展示推算区间而非单点值 |

### 4.4 推算值的报告撰写规范

```markdown
# ✅ 正确写法（完整推算标注）

泛联信息2024年营收约为 **~2,550万元(年化)** 🟡
（依据：硬氪报道"2024年营收超3倍增长"，以2023年约850万为基数，
 按3倍增速计；⚠️ "3倍"为公司方表述未经审计，实际值可能存在±30%偏差。

# ❌ 错误写法（各种违规）

❌ "泛联信息2024年营收约2,550万元"     → 缺推算方法和原始值
❌ "泛联信息2024年营收约2,553.72万元"   → 虚假精度（E-E3违规）
❌ "泛联信息营收是杉岩数据的0.51倍"     → 用🟠推算值做精确对比（E-E4违规）
❌ "预计2026年营收可达2.2亿"           → 二次推算+超长外推（E-E1+E-E5违规）
```

```markdown
|-----------|-----|------|------|------|--------|------|
| 注册资本 | 2,483.94万 | N/A | 企查查 | 静态事实 | 1.00 | — |
| 员工规模 | 100+人 | 🟢 | 百科+招聘网站 | 直接数据 | 0.85 | 2025年数据 |
| 2024年营收 | ~2,550万 | 🟡 | 硬氪+企查查 | 推算(增速外推) | 0.70 | 详见推算说明 |

**统计摘要**：
- 总数据点：42
- 🟢 FRESH：28 (66.7%)
- 🟡 CURRENT：10 (23.8%)
- 🟠 STALE：3 (7.1%)
- 🔴 OBSOLETE：1 (2.4%) — 仅用于历史基线
- 推算值：5 (11.9%)

```

### 5.3 质量分布的可接受阈值

| 指标 | 优秀 | 合格(最低门槛) | 不合格 |
|------|------|---------------|--------|
| 🟢+🟡 占比 | ≥ 85% | **≥ 70%** | < 70% |
| 🔴 用于定量判断 | 0 次 | **0 次** | ≥ 1次 |
| 推算值占比 | ≤ 20% | **≤ 35%** | > 35% |
| 完全缺口(核心PIR) | 0 个 | **0 个** | ≥ 1个 |
| 完全缺口(辅助PIR) | ≤ 1 个 | **≤ 2 个** | > 2个 |

> 低于"合格"门槛的报告必须在封面标注 **"⚠️ 数据质量受限报告"**。

### 6.3 对 `pir-template.md` 的增强

```

  🟢 FRESH  → 1.00 （不打折）

  🟠 STALE  → 0.75 （显著打折）
  🔴 OBSOLETE→ 0.50 （重度打折，且禁止用于CRITICAL PIR）

  PIR-BASIS 的证据强度本可打 0.90
  但核心数据（营收）来自 🟠 STALE 级别
  最终 effective_confidence = 0.90 × 0.75 = 0.675
  → 该 PIR 可能不满足 target_confidence 0.85
  → Agent 必须寻找更新的数据或使用推算补全
```

## 八、设计原则总结

| 原则 | 说明 |
|------|------|
| **闭环原则** | 任何数据需求都必须有输出（直接数据/推算/影响评估），不允许无理由跳过 |
| **透明原则** | 每个非直接数据都必须暴露其推导链，读者可自行判断可信度 |
| **分级原则** | 不是所有数据都需要同等精度——核心判断用高标准，辅助信息可放宽 |
| **保守原则** | 推算时宁可低估（给出更宽的不确定性区间），也不可过度自信 |
| **溯源原则** | 从报告正文的数值 → JSON的结构化记录 → 原始搜索结果，三层可追溯 |
