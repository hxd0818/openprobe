# 竞争对手调研方法 ⭐ v9.4
> **v9.4 核心变革**：从"发现竞对"升级为**"发现+深度拆解竞对关系"**。
> Agent 不仅要知道"谁在竞争"，更必须回答**"它们是否真的在同一赛道竞争"**这一根本问题。
> 
> **⭐v9.4 新增核心**：**六维深度竞对分析框架（§A）**——禁止仅用财务数据做简单对比，
> 必须从业态基因→发展阶段→选址策略→供应链壁垒→单店经济→组织治理六个维度逐层拆解。
> 
> 适用场景：所有行业、所有企业类型、所有商业模式。

## 1. 核心思维
### 为什么不能固化
| 固化方式 | 问题 |
|---------|------|
| 固定4条路径（v9.0） | 不管什么行业都跑同一套query |
| 固定路径+5套行业矩阵（v9.2） | 从1个模板变5个，仍是选择题 |
| **任何预设分类** | 无法应对真实世界的复杂性——同行业内不同商业模式差异巨大 |
### 真实世界的例子
同样是"零售业"：
- **KK集团**：购物中心大店集合店 → 竞对是其他集合店 + 抢商场位置的任何人
- **山姆会员店**：仓储式会员制超市 → 竞对是Costco/盒马/永辉，不是KKV
- **优衣库** | 基本款服装 → 竞对是ZARA/GAP/H&M，也不是KKV
同样是"SaaS"：
- **Salesforce**（通用CRM）→ 竞对是HubSpot/Dynamics/Zoho
- **Figma**（设计协作）→ 竞对是Sketch/Adobe XD/Canva
- **Notion**（知识管理）→ 竞对是飞书文档/Confluence/Obsidian
**同一"行业标签"下的企业，竞对可能完全不同。** 只有理解了具体的商业边界才能准确推导竞对。
### 新版框架
```
Agent 自主推理链:
  Step 1: 定义商业边界（"这家公司到底在做什么生意？"）
    → 卖什么？卖给谁？在哪卖？怎么赚钱？靠什么护城河？
    
  Step 2: 推导竞争全域（"还有谁在做类似的事？"）
    → 产品竞争 / 客户竞争 / 渠道竞争 / 模式竞争 / 注意力竞争
    → Agent 自己展开每个维度，不设上限
    
  Step 3: 规划发现策略（"我怎么找到这些玩家？"）
    → Agent 针对每个竞争域自己决定用什么方法
    → 工具箱仅供参考，不强求使用
    
  Step 4: 执行 + 动态调整
    → 搜索中发现新线索 → 追踪、回溯修正竞争域
    → 某方法无效 → 自行切换，不死守计划
    
  Step 5: 验证 + 反遗漏
    → 多源交叉确认
    → 系统性自检：龙头都在？细分覆盖？隐形巨头？
    
  Step 6: 分级 + 输出
    → L1-L4 四级分类
    → 全景图谱写入报告
```

## 2. Step 1：定义商业边界
> 在搜索任何竞对之前完成。基于 Phase 0 数据（百科+初始泛搜）即可启动。
### 商业边界五问
| # | 问题 | 回答后得到的信息 |
|---|------|-----------------|
| Q1 | **卖什么？**（产品/服务本质） | 确定产品竞争域的范围和边界 |
| Q2 | **卖给谁？**（客户画像） | 确定客户竞争域和注意力竞争域 |
| Q3 | **在哪卖？**（渠道/场景） | 确定渠道竞争域 |
| Q4 | **怎么赚钱？**（盈利模式） | 确定模式竞争域 |
| Q5 | **靠什么护城河？**（claimed moat） | 判断哪些竞对真正危险，哪些只是噪音 |
### 输出格式
```markdown
## [目标企业名] 商业边界定义
### 一句话定位
[≤30字，描述商业本质]
### 产品维度
- 核心品类：[具体]
- 品类宽度：[窄(单品类) / 中(相关多元) / 广(跨领域)]
- 自有 vs 第三方比例：[大致比例]
### 客户维度
- 目标客群：[年龄/性别/收入/心理特征/行为特征]
- 决策驱动：[功能 / 情感 / 社交 / 价格 / 关系]
- 复购依赖度：[高/中/低]
### 渠道维度
- 主力渠道：[线上/线下/混合/直销/代理/...]
- 线下具体形态：[如适用]
- 关键位置资源：[如"核心商圈点位"/"政府采购目录"]
### 盈利维度
- 主要收入来源：[具体]
- 成本结构敏感点：[租金/人力/研发/营销/原材料/...]
- 规模效应强弱：[强/中/弱]
### 护城河判断
- 已建立的优势：[具体]
- 虚假/脆弱的护城河：[如果有的话]
- 真正的壁垒：[如果有的话]
- ⚠️ 如果判断为"无明显护城河"——这本身就是重要发现
```

## 3. Step 2：推导竞争域
> 从 Step 1 的答案出发，**系统性地穷举**所有可能的竞争方向。
> 这是纯思维推演，不涉及搜索。目标是"想全"，而非"想准"（留待后续验证筛选）。
### 五维竞争域框架
| 维度 | 核心问题 | 推导逻辑 |
|------|---------|---------|
| **C1: 产品竞争域** | 还有谁卖类似的东西？ | 从Q1"卖什么"展开——同类/子集/超集/相邻品类 |
| **C2: 客户竞争域** | 同一批人还在把钱花在哪？ | 从Q2"卖给谁"展开——同人群不同品/线上替代/娱乐替代 |
| **C3: 渠道竞争域** | 还有谁在抢同一个"柜台"？ | 从Q3"在哪卖"展开——同商场/同商圈/同平台 |
| **C4: 模式竞争域** | 还有谁用类似的商业模式？ | 从Q4"怎么赚钱"展开——类似盈利逻辑的企业 |
| **C5: 注意力/预算竞争域** | 同一笔预算还可能流向哪？ | 综合(Q2+Q4)——所有争夺目标客户钱包份额的对手 |
### 展开要求
**每个维度至少列出 3 个推导方向**，多列不限：
```
示例（以KK集团为例）:
C1 产品竞争域:
  ├─ 同品类直接替代: 其他美妆/生活/潮玩集合店品牌
  ├─ 品品子集/超集: 大型综合零售商也含这些品类（屈臣氏/三福/沃尔玛）
  └─ 品品相邻扩展: 从美妆扩展到个护健康（屈臣氏优势领域）
C2 客户竞争域:
  ├─ 同客群不同品类: Z世代还买潮玩（泡泡玛特）、时尚（SHEIN）
  ├─ 同客群线上替代: 抖音/小红书/淘宝品牌旗舰店
  └─ 同客群娱乐/体验: 剧本杀/演唱会/旅游（预算分流）
C3 渠道竞争域:
  ├─ 同一商场主力店: 所有需要大铺位的租户
  ├─ 同一商圈争客流: 周边3公里内的体验型门店
  └─ 渠道资源竞争: 有限的核心优质商圈点位
C4 模式竞争域:
  ├─ 同样靠"选品+场景": MUJI/番茄口袋/1kg
  ├─ 同样靠连锁扩张: 其他加盟/直营连锁品牌
  └─ 同样靠网红引流: 其他打卡型零售
C5 注意力/预算竞争域:
  ├─ 线上分流: 全部电商平台/内容电商
  ├─ 跨界降维: SHEIN发力线下？
  └─ 新兴形态: 即时零售/社区团购等
```
### 关键原则
- **宁多勿少**：推导阶段不做排除，所有可能的竞争方向都列入
- **标注来源**：每条注明是从哪个商业边界要素（Q1-Q5）推出的
- **不知道就标`[需研究]`**：某维度确实想不出第3个方向时标注，Step 3 重点补搜

## 4. Step 3：自主规划发现策略
> **这是最核心的一步。Agent 根据 Step 1-2 的分析结果，自行决定针对每个竞争域用什么手段去发现竞对实体。**
### 规划要求
Agent 在开始竞对专项搜索前，**必须输出**：
```markdown
## 竞对调查计划
### 商业边界判断摘要
[2-3句话：这家公司的商业本质是什么]
### 推导出的竞争域概览
[C1-C5 各有哪些方向，总计多少个子方向待探索]
### 本轮次规划
**Round X（竞对专项）：**
- 目标竞争域：[本轮重点查哪个域]
- 选择的方法：[Agent自己决定的具体手段]
- 选择理由：[为什么认为这个方法对这个域有效]
- 预期产出：[希望获得什么信息]
```
### 自主规划的指导原则（非强制规则）
以下原则帮助 Agent 做出好的决策，但不限制发挥：
**原则1：从信息最密集的地方入手**
- 有招股书/年报的企业 → 先挖法定披露文件（同行对比章节=竞对名单）
- 有榜单的行业 → 先查榜单（一次获多家）
- 有线下门店 → 查商场入驻/加盟招商信息
- 有招投标 → 查中标公告（同场出现的=竞对）
**原则2：从已知向未知扩散**
- 已知一个竞对 → 搜"XX vs XX""XX 替代" → 发现更多
- 已知一个赛道 → 搜"XX行业 品牌 公司 排行" → 发现全貌
- 已知一个商场/渠道 → 查其入驻品牌 → 发现渠道竞对
**原则3：换角度重复搜索**
- 用产品角度搜一轮
- 用客户角度搜一轮（"Z世代喜欢的美妆品牌"）
- 用渠道角度搜一轮（"购物中心一楼美妆区品牌"）
- 用投资角度搜一轮（"XX赛道 融资 公司"）
- **每轮用不同的关键词组合，避免搜索盲区**
**原则4：关注"生态关联"**
- 发现一个竞对后，追问："它属于哪个更大的集团？集团下还有哪些品牌？"
- 这些"兄弟品牌"可能分别对标目标企业的不同产品线
- （如 WOW COLOUR ↔ 名创优品 ↔ TOP TOY = 同一体系三线围堵 KK）
### ❌ 禁止的行为
- ❌ 在没有完成 Step 1-2 的情况下直接开始搜索竞对
- ❌ 只使用搜索词中提到的名字作为竞对来源（那是循环论证）
- ❌ 连续2轮使用相同的角度/关键词组合而无新发现时不换思路
- ❌ 发现竞对后不追踪其背后的集团/资本关系

## 5. Step 4：执行与动态调整
### 每轮记录格式
```markdown
### Round X 执行记录
- **目标竞争域**: [本轮查的是 C1-C5 中的哪个]
- **使用方法**: [实际用了什么]
- **结果摘要**: [新发现了哪些竞对实体]
- **图谱更新**: [是否需要回溯修改 Step 2 的竞争域展开表]
- **下一步**: [还缺什么]
```
### 动态调整触发条件
| 触发场景 | 动作 |
|---------|------|
| 发现意料之外的重要竞对 | 追踪它（即使不在原计划中），回溯它属于哪个竞争域 |
| 发现竞对间的生态关联（同集团/同投资人） | 深挖整个生态网络，不只看单体 |
| 某方法连续2轮无新发现 | 换方法/换角度，不死磕 |
| 某竞争域已充分覆盖 | 标记为"satisfied"，转向未覆盖的域 |
| 发现全新的竞争维度（原Step 2未列出） | **立即更新 Step 2 竞争域展开表** |

## 6. Step 5：多源验证与反遗漏检测
### 6.1 交叉验证
将所有发现的竞对按**发现来源**统计出现次数：
| 实体名 | 来源数 | 典型来源 | 置信度 |
|--------|-------|---------|--------|
| 例: 名创优品 | 4+ | 榜单+对比+融资+商场 | 🔴极高 |
| 例: WOW COLOUR | 2 | 对比+融资 | 🟡高 |
| 例: 番茄口袋 | 1 | 仅对比 | 🟠需验证 |
**判定**：
- **≥3 个独立来源** → 直接分级入库
- **2 个来源** → 入库，建议补充一轮
- **1 个来源** → **必须**找第二个来源确认，或标记为"候选"
### 6.2 反遗漏自检清单
在宣布竞对发现完成前，**逐条回答**以下问题：
#### A. 行业龙头覆盖检查
```
□ 这个行业的全球/全国TOP5上市/非上市企业是谁？我的列表都有吗？
□ 这个行业的上市公司完整名单是什么？有没有漏掉的？
□ 最近一年获得大额融资（>1亿RMB）的公司有哪些？我的列表覆盖了吗？
```
**如果发现有遗漏 → 立即补搜并纳入**
#### B. 细分赛道覆盖检查
```
□ 回顾 Step 2 的 C1-C5 展开:
  - C1 的每个子方向都有对应竞对了吗？
  - C2 的每个子方向都有对应竞对了吗？
  - ...
  - 任何一个方向完全空白 = 可能存在遗漏 → 补搜
```
#### C. 隐形巨头检查
```
□ 有没有被大公司内部孵化/收购的品牌？（搜"XX 孵化 子品牌 项目 投资"）
□ 有没有通过子公司/合资/代运营方式参与这个赛道的巨头？
□ 有没有体量巨大但品牌名不含行业关键词的公司？
```
#### D. 生态关联检查
```
□ 每个L1竞对背后是否有更大的企业集团？
□ 集团内是否有其他品牌也在不同维度与目标企业竞争？
□ 是否存在"一个老板打了目标企业三个品牌"的情况？
```

## 7. Step 6：分级与全景输出
### 7.1 四级分类标准
| 级别 | 定义 | 判定标准 | 报告要求 |
|------|------|---------|---------|
| **L1 直接竞对** | 同赛道+同产品+同客户，重叠度>70% | 独立分析段落 ≥300字 | 含生态关联分析 |
| **L2 间接竞对** | 不同产品但抢同客户/同渠道/同注意力 | 竞争定位 ≥2句话 | — |
| **L3 潜在威胁** | 大厂跨界/新兴玩家/有能力进入 | 威胁评估 ≥1句话 | — |
| **L4 替代品** | 非同类但解决同一需求 | 列表提及即可 | — |
### 7.2 分级决策树
```
发现实体E
  │
  ├─ E的产品与服务 和目标 重叠度 > 70%？
  │   └─ 是 → L1
  │   └─ 否 ↓
  │
  ├─ E的目标客户 和目标 重叠度 > 50%？
  │   └─ 是 → L2（客户竞争型）
  │   └─ 否 ↓
  │
  ├─ E的渠道资源 和目标 有直接竞争？
  │   └─ 是 → L2（渠道竞争型）
  │   └─ 否 ↓
  │
  ├─ E有能力且有可能进入目标赛道？
  │   └─ 是 → L3
  │   └─ 否 ↓
  │
  └─ E能满足目标客户同类需求？
      └─ 是 → L4
      └─ 否 → 不列入（记录原因）
```
### 7.3 最终输出格式
报告中的竞争格局章应包含：
1. **商业边界定义摘要**（1段话）
2. **竞对全景图谱表**（L1-L4 全列表，含发现来源数和置信度）
3. **L1 独立深度画像**（每家≥300字，含生态关联）
4. **梯队划分图**（领导者/挑战者/利基/新进入者）
5. **竞争矩阵**（多维对比，维度根据行业自适应）
6. **差异化分析与威胁研判**
7. **PIR-COMPETITION 履行声明**

## 8. L1 深度画像要求
每家 L1 必须包含以下全部要素：
### 8.1 必填项
| 要素 | 要求 | ⚠️注意 |
|------|------|--------|
| 身份信息 | 全称/简称/成立时间/性质/总部/股权/**实控人（穿透到自然人）** | 不要只写公司名不写控制人 |
| 产品与服务 | 逐条产品线 + 与目标的重叠度分析 | 必须明确指出哪些线在直接竞争 |
| 优劣势 | 各3-5条 + **每条附证据来源** | 禁止主观臆断 |
| 市场地位 | 排名/份额/融资/IPO/动向 | 数据标注精确/推算 |
| **生态关联** | ⭐ 是否属于更大集团？协同品牌？ | 如 WOW COLOUR↔名创优品↔TOP TOY |
### 8.2 最低篇幅
- L1：**≥300字**独立段落（非表格形式）
- 构成建议：定位规模(80字) + 竞争态势(120字) + 生态关联(50字) + 战略预判(100字)

## 9. 竞争格局综合分析
### 9.1 梯队划分
```
第一梯队（领导者）：市场份额大 + 品牌强 + 覆盖广 + 通常已上市
第二梯队（挑战者）：细分有优势，整体追赶，通常冲刺IPO
第三梯队（利基玩家）：专注特定场景/区域/客户群
第四梯队（新进入者/替代品）：新兴/跨界/非同类
```
### 9.2 差异化分析三问
1. **目标企业的独特价值主张？**（别人做不到或不做的）
2. **各竞对的护城河？**（IP/技术/客户关系/牌照/网络效应/规模/生态）
3. **市场空白点？**（没人做好或没人做的）
### 9.3 威胁研判
特别关注**体系化围堵**风险（v9.3新增）：
| 威胁类型 | 信号 | 紧迫程度 |
|---------|------|---------|
| 体系化围堵 | 一个企业集团通过多个子品牌分别在目标企业的各产品线部署竞对 | 🔴 最高 |
| 直接竞争 | L1竞对融资/上市/扩张 | 🔴 高 |
| 降维打击 | 大厂跨界 | 🟡 中 |
| 渠道挤压 | L2渠道竞对争夺有限资源 | 🟡 中 |
| 替代品崛起 | 新技术/新模式 | 🟡 低-中 |

## ⭐ A. 六维深度竞对分析框架（v9.4 — 强制执行）

**核心规则**：禁止仅用财务绝对值（营收/利润/用户数/市值）做竞对强弱判定。
对比前必须先完成六维拆解，确认"是否在同一赛道竞争"。

**行业自适应**：六维的分析目的通用，但具体检查项和指标由 Agent 根据目标行业自行适配。
本文件中的表格为思考模板，非限定性答案。

### A.1 前置检查（⭐ 必须最先完成）

对比前先回答：**二者是否同一赛道？**
- 短答案：[是/否/部分重叠/表面重叠本质不同]
- 一句话类比：[如 "A vs B ≈ 精品咖啡 vs 星巴克"]
- 不用数字如何描述关系？

**三禁**：
- ❌ 未完成六维拆解 → 禁止输出"X是Y的N%"
- ❌ 禁止仅用绝对值判定强弱
- ❌ 禁止忽略阶段差异直接比

### A.2 六维拆解框架（行业自适应版）

#### 维度一：商业本质（Business Essence）⭐⭐⭐

**分析目的**：回答"这两家公司到底在卖什么？解决什么问题？靠什么赚钱？"

**通用检查模板**（Agent 根据行业适配具体指标）：

| 检查方向 | 目标企业 | 竞对X | 差异性质 |
|---------|---------|-------|---------|
| 本源定位（一句话） | | | |
| 解决的核心问题 | | | 问题不同？ |
| 目标客户（谁买/谁用） | | | 客群错位？ |
| 价值主张（为什么选你） | | | 价值主张差异？ |
| 决策动因（怎么决定用你） | | | 动因本质差异？ |
| 交付形态（产品/服务/平台/内容） | | | 形态对立？ |

**关键产出**：判断二者是**同一物种**还是**不同物种**。

> 不同商业本质类型必须明确标注并调整后续参照系。

#### 维度二：发展阶段与时间线（Development Stage）⭐⭐

**分析目的**：确保公平对比——处于不同阶段的企业不应直接用绝对值比较。

| 时间节点 | 目标企业 | 竞对X | 阶段差距 |
|---------|---------|-------|---------|
| 成立年份 | | | 相差__年 |
| 首次商业化/首客/首品 | | | 相差__年 |
| 规模里程碑（因行业而异） | | | 各自用了__年 |
| 当前阶段判断 | □种子期 □成长期 □成熟期 □转型期/衰退期 | □同上 | 是否可比？|

> 阶段对标：竞对成立更早时，应使用其"N年前同期"数据作参照，而非当前最新数据。

> 各阶段特征：种子期→成长期→成熟期→转型/衰退期；从核心市场/扩张/品牌技术/资本四方面描述。

#### 维度三：获客与渠道（Go-to-Market & Channel）⭐⭐

**分析目的**：企业如何触达客户、获取订单？渠道策略决定增长天花板和抗风险能力。

**通用检查模板**（⚠️ Agent 必须根据行业替换为恰当指标）：

| 检查方向 | 目标企业 | 竞对X |
|---------|---------|-------|
| **获客方式** | [线上/线下/直销/渠道/生态/招投标/...] | |
| **渠道结构** | [单一渠道 vs 多渠道 / 自有 vs 第三方] | |
| **客户集中度** | [前N大客户占比 / 是否依赖少数大客户] | |
| **地域覆盖** | [本地/区域/全国/全球 / 一线城市vs全域/...] | |
| **获客成本(CAC)趋势** | [上升/下降/稳定] | |
| **渠道依赖风险** | □低 □中 □高 □极高 | □同上 |

> Agent 自主判断：①触达路径是什么？②是否单一依赖(>60%即危险)？③有无替代？
> （示例：零售=商场依赖/SaaS=PLG依赖/制造=大客户依赖/金融=牌照依赖）

> 高风险信号：单一来源>60% / 渠方可随时切断 / 已发生解约事件 / 核心资源被竞对抢占

#### 维度四：核心能力与壁垒（Core Capability & Moat）⭐

**分析目的**：企业的护城河是什么？是"个人能力依赖型"还是"结构性壁垒"？这决定了竞争优势的可持续性。

**通用检查模板**：

| 检查方向 | 目标企业 | 竞对X |
|---------|---------|-------|
| **核心能力类型** | [技术/品牌/网络效应/规模/牌照/资源/数据/...] | |
| **壁垒的可复制性** | □极易 □较难 □很难 □几乎不可复制 | □同上 |
| **研发/投入强度** | [研发投入/专利数/人才密度/...] | |
| **供应链/合作伙伴控制力** | | |
| **核心毛利来源** | [溢价来源：技术/品牌/稀缺性/位置/...] | |

**壁垒分级（通用版）**：

| 等级 | 类型 | 可复制性 | 通用例子 |
|------|------|---------|---------|
| L4 极强 | 网络效应/垄断性牌照/标准制定权 | 几乎不可复制 | 微信社交网络/支付牌照/通信标准 |
| L3 强 | 规模效应+全球布局/深度技术积累 | 需巨量资本+多年 | 台积电制程/空客供应链/Google AI |
| L2 中 | 品牌认知/细分领域专长/特定客户关系 | 需多年积累但可追赶 | 区域品牌/SaaS垂直赛道/咨询关系 |
| L1 弱 | 无明显壁垒/纯执行力/易替代品 | 易被复制模仿 | 普通贸易/同质化SaaS/无差异服务 |

**【行业适配】** Agent 根据目标企业所在行业判断其壁垒属于哪一级：
- *科技*：专利组合/工程师密度/数据飞轮/开发者生态
- *零售*：买手选品/IP授权/供应链规模/点位资源
- *制造*：工艺know-how/设备产能/认证资质/成本曲线
- *金融*：牌照/风控模型/资金成本/用户数据/场景嵌入
- *医疗*：临床数据/监管审批/医生关系/专利药

#### 维度五：经济模型与效率（Unit Economics & Efficiency）⭐⭐

**分析目的**：绝对值不具可比时，效率和健康度才是真指标。找到该行业**最核心的效率度量**进行公平比较。

**通用检查模板**：

| 检查方向 | 目标企业 | 竞对X | 公平比较方式 |
|---------|---------|-------|------------|
| **核心产出单元** | [订单/用户/门店/项目/席位/...] | | ⚠️ 单元定义不同时不可直接比 |
| **单元经济效率** | [该行业最核心的效率指标] | | ✅ **最核心的公平比较指标** |
| **利润率水平** | [毛利率/净利率/EBITDA率] | | 同模式下可比 |
| **投资回收周期** | | | 含不同商业模式视角 |
| **成本结构特征** | [固定/变动/杠杆] | | 重/轻资产 |
| **抗风险能力** | [收入波动敏感度] | | 弱/中/强 |

**【Agent 自主选择方法】**

Agent 应根据目标企业所在行业，**自行判断**以下三个问题来决定用什么指标：

1. **这个行业的企业靠什么单元产出价值？**
   → 这个单元就是比较的基础（单店/用户/订单/项目/席位/工厂/...）

2. **什么指标能反映这个单元的真正效率？**
   → 不是绝对值（总营收/总用户），而是比率或单位效率
   → 思考方向：投入产出比 / 利用率 / 利润率 / 回报周期

3. **两家企业的这个单元是否可比？**
   → 如果单元定义、量级、模式差异大 → 必须先标准化再比较
   → 如果连单元本身都不同（如平台 vs 工具）→ 应说明不可直接比的原因

> **示例（仅演示思考过程，非限定性答案）：**
> - 零售业 Agent 可能选：坪效（因为面积差异大）
> - SaaS Agent 可能选：LTV/CAC（因为获客模式差异大）
> - 制造业 Agent 可能选：产能利用率（因为设备投入差异大）
> - 金融 Agent 可能选：ROA（因为资产结构差异大）
>
> **关键是 Agent 自己的推理过程，不是套用现成答案。**

> **效率铁律**：产出单元定义/量级/模式差异显著时 → 禁止直接比绝对值 → 必须折算为效率指标后再比。这是所有行业的基本常识。

完成 A.2 六维拆解后，Agent 必须**重新定义**目标企业与每家 L1 竞对的关系：

#### A.3.1 交集领域判定表（行业自适应版）

| 交集领域 | 交集程度(0-5星) | 具体说明 |
|---------|----------------|---------|
| **产品/服务重叠** | ★____☆ | [功能/能力的重叠程度] |
| **目标客户重叠** | ★____☆ | [决策者/使用者/付费者的重叠] |
| **获客渠道/场景重叠** | ★____☆ | [在哪里争夺同一批客户] |
| **关键资源争夺** | ★____☆ | [供应链/人才/牌照/数据/点位/...] |

|---------|------------|---------------------|

#### A.3.3 真正的竞对坐标系（三圈层）

**第一圈层（直接竞对）**：同类商业本质 + 同类客户 + 同类获客渠道 → 用 A.2 全部六维逐项对比

**第二圈层（间接竞对）**：不同产品但抢同一批客户的**钱/时间/注意力/预算**

**第三圈层（隐藏最大威胁）**
> 最大敌人可能不是某家公司，而是**行业结构性变迁**（技术替代/消费习惯变化/基础设施更替/政策转向）。这个问题的答案往往比竞对分析更有价值。

### A.5 执行时机与强制要求

| 要求 | 说明 |
|------|------|
| **触发条件** | 每发现一家 L1 级竞对时**必须**执行 |
| **最低要求** | 六维全部填写，不允许跳过任何维度 |
| **行业自适应** | 每个维度的具体检查项和指标**必须**根据目标企业所在行业自行适配，不得照搬其他行业的指标 |
| **禁止事项** | 未完成六维拆解前不得输出"X是Y的N%"类结论；不得跨行业套用不适用的指标 |
| **与旧版的关系** | 本框架是对 §9（竞争格局综合分析）的**升级替代** —— §9 三问法保留为快速初筛，L1竞对必须走完 A.1-A.4 全流程 |
| **篇幅要求** | 每家 L1 竞对的深度分析 ≥ **800字**（从原300字上调） |


*本文件 v9.4 的设计哲学：**Agent 是分析师，不是脚本执行者。***
***不预设行业分类，不限定搜索路径，不规定工具使用顺序。***
***Agent 自主理解 → 自主推导 → 自主规划 → 自主执行 → 自主验证。***
***Skill 文件的职责是教会Agent如何思考，而不是告诉它做什么。* *
