## 1. 报告 vs 数据的区别

| | 数据（JSON） | 分析报告（MD/PDF） |
|--|-------------|-------------------|
| 目的 | 系统使用、可查询、可复用 | 人阅读、做决策参考 |
| 形式 | 结构化、完整、精确 | 叙述性、有观点、有逻辑链 |
| 内容 | 所有发现的原始记录 | **经过提炼的洞察与判断** |
| 缺失数据 | 留空/null | 标注"未获取到该数据"，**不中断分析** |

## 2. 写作铁律

1. **每段必须有观点** — 不写"XX公司成立于2015年"这种纯事实罗列
   - ❌ "名创优品2025年营收214亿，8485家门店"
   - ✅ "名创优品规模达KK集团的6倍（214亿 vs 35.5亿），其旗下TOP TOY正以94.8%的增速直接冲击X11的核心市场"

2. **数据服务于结论** — 先给结论，再用数据支撑
   - ❌ 先列一堆表格再总结
   - ✅ "KK集团面临'一超多强'的竞争格局，名创优品以6倍规模优势形成全方位压制"

3. **缺失数据不阻断分析** — 没找到的数据标注后继续推理
   - ❌ 因缺具体供应商名单而跳过上游分析
   - ✅ "具体供应商名称未能在公开渠道获取（⚠️ 信息缺口），但从买断制模式可推断..."

4. **贯穿全文的分析主线** — 报告必须回答一个核心问题
   - 主线示例："这家公司在行业中处于什么位置？面临什么风险？机会在哪？"
   - 每个章节都应回扣这条主线

5. **判断要有依据** — 每个分析性结论必须标注证据来源或置信度
   - ✅ "判断为L1直接竞对（证据：双方产品高度重叠+招股书互为对标）[置信度:高]"

## 3. 报告结构（分析版）

```
# {企业} 市场竞争格局调研报告

## 执行摘要（最重要的部分，200-300字）
   一句话定位 + 核心发现(3-5条) + 关键风险 + 总体判断
   读者只看这一段就能了解80%的结论

## 1. 企业定位与战略画像（不是基本信息表）
   它是谁不重要 → 它选择做什么、怎么做、凭什么赚钱才重要
   · 商业模式本质（一句话）
   · 在价值链中的位置
   · 规模与阶段判断（成长期/成熟期/衰退期）
   · 高管团队背景与战略风格（⭐v9新增，见people-research.md）

## 2. 竞争格局（核心章节，占报告40%+篇幅）
   ### 2.1 格局总判（先给结论）
       一句话定义格局形态（"一超多强"/"双寡头"/"碎片化"）
   ### 2.2 主要威胁深度分析（逐个分析，不是表格）
       每个L1竞对一段：
       · 为什么它是最大威胁？（具体分析，不是列数据）
       · 双方的正面战场在哪？
       · KK有什么应对筹码？
   ### 2.3 竞对高管对比（⭐v9新增，见people-research.md第4节）
       目标CEO vs 各L1 CEO的背景差异 + 解读
   ### 2.4 竞争态势演变（动态视角）
       这个格局在怎么变？未来1-2年可能发生什么？

## 3. 产业链地位分析
   不是供应商/客户名单 → 是对议价能力和风险的判断
   · 上游：谁有话语权？被卡脖子的可能性？（具名供应商分析）
   · 下游：客户依赖度？渠道可持续性？（具名客户分析）
   · 集中度风险意味着什么？

## 4. 资本与健康度
   不是融资历史列表 → 是对生存状态和持续能力的判断
   · 能活多久？钱够烧吗？
   · IPO前景如何？为什么多次递表失败？
   · 股东结构带来什么影响？
   · 关键投资人意图分析（⭐v9新增，见people-research.md第5节）

## 5. 综合研判与建议
   ### 5.1 核心判断（3-5条带依据的结论）
   ### 5.2 风险排序（按紧迫性和严重性排列）
   ### 5.3 关键监控指标（后续应该跟踪什么）

## 附录：数据来源与置信度说明
   （不是报告主体，是支撑材料）
```

## 4. 建议篇幅分配（6000字基准）

| 章节 | 建议字数 | 占比 | 说明 |
|------|---------|------|------|
| 执行摘要 | ~300字 | 5% | 核心判断浓缩 |
| 企业画像（含高管团队+联系方式） | ~900字 | 14% | ⭐含高管背景段落 + **对外联系方式专段** |
| **竞争格局** | **~2500字** | **42%** | **L1逐个深度+L2综合+高管对比+格局演变** |
| 产业链（具体实体分析） | ~1200字 | 20% | 具名供应商/客户/合作伙伴 |
| 资本与健康度（含投资人意图） | ~800字 | 13% | ⭐含投资人意图分析 |
| 综合研判 | ~400字 | 7% | 结论+风险+监控指标 |
| **合计** | **~6000字** | **100%** | 浮动±20%可接受 |

> 竞争格局必须占40%以上。如某维度信息极度匮乏，可将篇幅重新分配给其他维度，但需在执行摘要中说明。

## 5. 表格使用原则

- **报告中的表格 ≤ 3 个**（只在确实需要对比时使用）
- 表格必须带分析行（"意味着什么"）
- 禁止纯信息罗列表格（"成立时间/法人/注册资本"这类放附录）
- 竞争矩阵可以用，但必须附带解读段落

## 6. 禁止出现的写法

- ❌ "根据搜索结果..." / "通过百度搜索发现..." （方法论描述不属于报告内容）
- ❌ 大段原文摘录（除非极关键且无法转述）
- ❌ "详见XXX.json" （报告自包含，不引用外部文件）
- ❌ 缺失数据留空白不解释（必须说明影响）
- ❌ 每个实体平均分配篇幅（重要实体多写，次要的一笔带过）

## 7. §11.6 📐 数据对比铁律 — 跨实体比较的强制性规则 ⭐⭐（v9.1新增）

> **触发条件**：报告中出现任何两个及以上实体的**同类指标对比**（营收、门店数、利润、增速、市占率等）
> **违反后果**：报告质量判定为**不合格**，必须修正后重新生成

### 铁律一：时间口径必须对齐

| 规则 | 说明 | 示例 |
|------|------|------|
| **同财年优先** | 对比双方必须使用相同或最接近的完整财年数据 | KK用2023全年 vs 名创用2023全年 ✅ |
| **禁止跨年裸比** | 禁止用A的2023年前10月 vs B的2025年全年直接算倍数 | ❌ "名创是KK的5倍"（47.7亿前10月 vs 214亿全年） |
| **非同期必须标注** | 如果确实无法获取同期数据，必须在对比处明确标注时间差异 | ✅ "名创优品2025年全年营收214亿元，KK集团最新披露为2023年前10月47.7亿元（⚠️时间口径不同，不可直接换算倍数）" |
| **部分年份数据必须说明** | 使用"前N个月""上半年"等非全年数据时，必须标注且不得用于计算精确比值 | ❌ "按前10月推算全年约57亿"；✅ 仅描述为"2023年前10月营收47.7亿元" |

### 铁律二：数据来源必须追溯

每次在报告中引用数值型数据时，Agent 必须在写作时内心（或注释中）确认：

```
数据引用自检模板：
┌─────────────┬──────────┬──────────┬──────────────┐
│ 指标         │ 数值      │ 时间范围   │ 来源          │
├─────────────┼──────────┼──────────┼──────────────┤
│ KK集团营收   │ 47.7亿   │ 2023前10月 │ 招股书(四次递表) │
│ 名创优品营收  │ 214亿    │ 2025全年   │ 2025年度财报    │
│ 对比结论     │ —        │ ⚠️不可直比  │ → 标注差异即可   │
└─────────────┴──────────┴──────────┴──────────────┘
```

### 铁律三：对比结论的措辞规范

| 场景 | 禁止写法 | 正确写法 |
|------|---------|---------|
| 时间不同 | "A是B的N倍" | "A规模显著大于B，但两者数据披露时间不同（A:2025全年 vs B:2023前10月），精确比值无意义" |
| 口径不同 | "A超过B" | "从各自最新披露看，A处于更高量级（具体见下表），但需注意统计口径差异" |
| 一个全年一个非全年 | 任何倍数/百分比对比 | 定性描述："A明显领先于B"，另起一行标注数据时间差异 |

### 铁律四：profile.json 数据必须带时间戳

所有 JSON 产出物中的数值字段，新增 `_data_period` 字段：

```json
// profile.json scale 字段示例
"scale": {
  "revenue": {"value": "47.7亿元", "period": "2023年前10月", "source": "招股书(2024.1递表版)", "note": "非全年数据"},
  "employees": {"value": "约1150+家门店", "period": "2025.12", "source": "百科+新闻交叉验证", "note": null}
}

// competitors.json 中竞对数据示例
"scale_snapshot": {
  "company_a": {"revenue": "214亿", "period": "2025全年", "source": "年度财报"},
  "company_b": {"revenue": "47.7亿", "period": "2023前10月", "source": "招股书"}
}
```

### 铁律五：报告中的对比表格必须含时间列

任何包含多实体数值对比的表格，**必须**包含"数据期间"列：

```markdown
| 企业 | 营收 | 数据期间 | 来源 |
|------|------|---------|------|
| 名创优品 | 214亿元 | 2025全年 | 年度财报 |
| KK集团 | 47.7亿元 | 2023前10月 | 招股书 |
```

> **没有"数据期间"列的对比表格 = 自动不通过**

### 执行机制

1. **采集阶段**：每个搜索到的数值数据，记录到对应JSON时必须带 `period` + `source`
2. **撰写阶段**：Agent 在写每一段对比文字前，先检查双方数据的 `period` 是否一致
3. **不一致时的处理流程**：
   - 尝试搜索缺失方的同期数据 → 如找到则使用同期数据
   - 如确实找不到 → 用定性语言替代定量比值 + 明确标注差异
4. **自检阶段**：在 11.7 自检清单中新增数据一致性检查项（见下方）

## 8. §11.7 报告发布前强制自检清单 ⭐（v9新增）

> 生成最终报告前，**必须逐项检查**以下清单。任一检查项不通过应先修正再发布。

### 8.1 数据一致性（⭐⭐v9.1新增 — 数据对比铁律执行检查）

- [ ] 报告中**所有跨实体数值对比**是否满足 §11.6 数据对比铁律？
- [ ] 每个对比是否包含**数据期间/财年**信息？（无时间列的对比表格 = 不通过）
- [ ] 是否存在**不同口径数据的直接倍数/百分比计算**？（如"是N倍""增长N%"涉及跨年数据）
- [ ] 所有 JSON 产出物中的数值字段是否带 `_data_period` 或 `period` 字段？
- [ ] 如存在不可比数据，是否已用**定性语言替代定量比值** + **明确标注差异**？

### 8.2 PIR 履行检查（⭐⭐v9.1新增 — 见 pir-template.md）

- [ ] 是否在 Phase 0 定义了完整的 PIR 列表（Layer 0用户指定 + Layer 1基线 + Layer 2实例化）？
- [ ] 所有 CRITICAL PIR 是否已达到 target_confidence？
- [ ] 所有 HIGH PIR 是否已达到 target_confidence 或标注 stalled/abandoned/n/a？
- [ ] 是否执行了"最后验证轮"？
- [ ] 报告中是否包含 PIR 履行情况摘要表？
- [ ] 如有未满足 PIR，是否在报告中明确标注并说明影响？

### 8.3 分析师职业规范（⭐⭐v9.1新增 — 见 analyst-identity.md）

- [ ] 是否已读取并内化 `references/analyst-identity.md` 的身份定义？
- [ ] 报告是否存在 **E1-E5 致命错误**？（存在任一 = 报告不合格）
- [ ] 报告是否存在 **E6-E10 严重错误**？（存在需修正后重新发布）
- [ ] 事实/推断/观点是否正确区分标记？
- [ ] 报告末尾是否包含分析师声明？

### 8.4 数据完整性

- [ ] entity_pool 中所有 confidence > 0.7 的实体是否都在报告中被引用？
- [ ] 所有 L1 竞对是否都有独立分析段落？
- [ ] 重点 L2 竞对是否至少有定位描述？
- [ ] 高管/关键人物是否有人物画像段落？

### 8.5 分析深度

- [ ] 竞争格局章节是否占全文40%以上？
- [ ] 是否有前瞻性判断（未来12-24个月演变）？
- [ ] 缺失数据是否都有⚠️标注+影响推理？
- [ ] 判断结论是否有证据来源标注？

### 8.6 格式规范

- [ ] 表格数量 ≤ 3？每个表格是否有解读段落？
- [ ] 是否存在"根据搜索结果发现"等方法论描述？
- [ ] 是否存在大段原文摘录？
- [ ] 执行摘要是否 ≤ 300字且包含核心判断？

### 8.7 数据时效性与完整性（⭐⭐v9.3升级 — 见 `data-quality-gate.md`）

> 完整规范（四级递进流程/推算方法/禁止清单/违规判定）见 `references/data-quality-gate.md`。
> **⭐v9.3 核心变更**：从"事后标注"升级为"事前预防"——新增四级递进(L1-L4)检查和时间锚定query检查。

#### 四级递进合规检查（⭐v9.3 新增）
- [ ] 每个量化数据点是否已执行 L1（带年份锚定搜索，≥2次）？
- [ ] L1 无结果时是否执行了 L2（碎片数据 hunt，≥3次）？
- [ ] L2 无碎片时是否执行了 L3（旧数据+推算因子）？
- [ ] 是否存在跳级直接使用旧数据的情况？（V-1/V-2 违规 = **不通过**）
- [ ] round_log 中是否有记录每级搜索的 query 和结果摘要？

#### 时间锚定 Query 检查（⭐v9.3 新增）
- [ ] 所有涉及量化数据的搜索 query 是否包含时间锚定？（年份/最新/年报）
- [ ] 是否存在无时间锚定的量化数据搜索？（Q5 违规 = 需补搜）

#### 时效分级标注检查
- [ ] 报告中所有数值型数据是否已标注时效/推算等级(🟢🟡/~碎片/~旧推算🟠/N/A)？
- [ ] 关键判断依赖的核心数据是否 ≥ 🟡 CURRENT 或 ~碎片推算？
- [ ] 是否存在裸用 🟠 STALE 数据（无~标记、无推算说明）？（= **不通过**）
- [ ] 🔴 OBSOLETE 数据是否仅用于历史基线描述？
- [ ] 基础事实豁免是否合理？（不在豁免列表内的被标记N/A = 不通过）

#### 缺口处理检查
- [ ] 所有标记"未获取"/null 的关键数据是否已走完四级递进（L1→L4）？
- [ ] 是否存在无理由跳过某一级的数据项？
- [ ] 完全缺口是否已评估对核心判断的影响？gap_items 是否记录到 status.json？

#### 推算合规检查
- [ ] 所有推算值是否附带：原始值 + 推算方法 + 不确定性说明 + 碎片/因子来源？
- [ ] 推算等级是否正确标注？（~碎片🟡 vs ~旧推算🟠，不可混同）
- [ ] 是否存在二次推算？（E-E1 违规 = **不通过**）
- [ ] 是否存在虚假精度？（E-E3 违规 = **不通过**）
- [ ] 低可信度推算(~旧推算🟠/≈)是否避免了跨实体精确对比？（E-E4 违违规）
- [ ] ~旧推算值(🟠)的可信度是否 ≤ 0.65 上限？

#### PIR 时效折扣检查（⭐v9.3 新增）
- [ ] 基于🟢/🟡数据的 PIR 是否使用了正确系数（×1.0 / ×0.95）？
- [ ] 基于~碎片推算数据的 PIR 是否使用了 ×0.85 系数？
- [ ] 基于~旧推算数据的 PIR 是否使用了 ×0.65 系数？
- [ ] 折扣后 effective_confidence < target_confidence 的 PIR 是否已标注为受限项？

#### 数据质量概览
- [ ] 报告附录是否包含"数据质量概览表"？
- [ ] 🟢+🟡+~碎片 占比 ≥ 70%？~旧推算占比 ≤ 25%？核心PIR完全缺口 = 0？
- [ ] 如低于阈值，报告封面是否已标注 **"⚠️ 数据质量受限报告"**？

### 8.8 质量门禁确认

- [ ] 迭代轮次 ≥ **6**？（否则标注 **"初步报告"**；⭐v9.5 从4上调至6，与 methodology.md C4 一致）
- [ ] entity_pool 实体数 ≥ 20？（否则标注 **"浅层调查"**）
- [ ] L1竞对全部完成画像？
- [ ] 数据质量门禁 v9.3 全部通过？（见上方 §8.7 区块，含四级递进+时间锚定+折扣检查）

> 自检结果写入 `status.json.final_quality_check`，作为报告发布的门禁记录。
